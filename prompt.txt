You are helping refactor an existing codebase called lilbot.

The goal is to convert lilbot into a modular framework for running local LLM agents.

This framework should resemble the architecture used by modern agent runtimes (such as OpenClaw-style systems), where an agent loop orchestrates LLM reasoning, tool usage, and memory.

Do NOT build a single chatbot application.

Instead build a reusable agent framework with clean modules.

-----------------------------------------------------

CORE OBJECTIVE

lilbot should become a framework for building autonomous agents that can:

- run locally
- call tools
- retrieve documents
- maintain memory
- execute multi-step tasks

The system should support an "agent loop":

User request
→ LLM reasoning
→ decide tool
→ execute tool
→ return result
→ continue until task complete

-----------------------------------------------------

PROJECT STRUCTURE

Refactor the codebase into something like:

lilbot/
    agent/
        runner.py
        planner.py
        context.py
    tools/
        registry.py
        base_tool.py
        shell.py

        index.py

    llm/
        provider.py
        prompt_builder.py
    cli/
        main.py

-----------------------------------------------------

AGENT RUNTIME

Create an AgentRunner class responsible for:

- managing the agent loop
- building prompts
- deciding when to call tools
- tracking intermediate state

Pseudo behavior:

while not task_complete:
    build prompt
    call LLM
    parse tool decision
    execute tool
    append result to context

-----------------------------------------------------

TOOL SYSTEM

Implement a tool registry.

Tools must include:

- name
- description
- input schema
- run() function

Example:

class Tool:
    name
    description
    run(input)

Tools should register automatically.

Example tools:

search_docs
shell_command
summarize_text

-----------------------------------------------------

RETRIEVAL (RAG)

Add support for document retrieval.

The system should:

- load documents
- embed them
- store embeddings
- retrieve top-k chunks

Used by the "search_docs" tool.

-----------------------------------------------------

LLM ABSTRACTION

Create a provider interface so lilbot can use local models

All LLM calls should go through:

llm.generate(prompt)

-----------------------------------------------------

MEMORY

Add two types of memory.

Conversation memory:
short-term context for the agent loop.

Vector memory:
long-term storage for documents and notes.

-----------------------------------------------------

SAFETY

Agents must not run arbitrary commands.

Add:

- tool allowlist
- logging of tool execution
- optional user confirmation for risky tools

-----------------------------------------------------

CLI INTERFACE

Provide a simple CLI.

Example:

lilbot run

User types a request.

Agent decides:

- search docs
- run tool
- summarize answer

-----------------------------------------------------

DESIGN PRINCIPLES

- modular
- minimal dependencies
- easy to extend with new tools
- easy to plug in different LLMs
- clean separation of concerns

Generate:
